# UI/UX Issue-Spotting Checklist (UNO-A)

## Core flows (must)
- Onboarding: 가입/로그인/인증(이메일/OTP/본인인증) 단계 최소화, 실패시 복구(재시도/고객센터/대체 로그인)
- Consent UX: 약관/개인정보/마케팅 동의가 명확히 분리되어 있고, "필수/선택"이 UI로 구분됨
- DM/채팅: 메시지 전송 실패/재시도/중복전송 방지, 로딩/오프라인 상태 표시
- Wallet/DT: 잔액/충전/사용/환불/정산 상태를 사용자가 오해하지 않게 표기("크레딧/포인트" 성격 명확)
- Funding: 티어 선택→결제→성공/실패(환불) 플로우가 1)명확 2)추적 가능 3)오류시 복구 가능
- Settings: 계정삭제/데이터 삭제(또는 탈퇴) 경로 존재, 보안(비번 변경/세션관리) 노출

## Friction & trust
- Price/fee transparency: 결제 전 "총액/수수료/환불조건"이 단일 화면에 요약됨
- Status visibility: 결제/환불/정산/검수(펀딩 승인) 상태가 명확한 라벨과 타임라인으로 제공
- Error handling: 네트워크/서버 오류 메시지가 행동 가능한 형태(다시 시도/문의/로그 참조키)
- Empty states: 최초 진입 시 빈 화면 가이드("무엇을 해야 하는지") 제공

## Accessibility & UI consistency
- 터치 타겟(최소 44px), 대비(텍스트 가독), 폰트 스케일, 스크린리더 라벨
- 색으로만 상태 전달하지 않기(아이콘/텍스트 병행)
- 다크/라이트 모드에서 동일한 의미 유지(경고/성공/CTA 색 정의)
- 로딩 스켈레톤/진행 표시가 일관됨

## Privacy by UX
- 민감정보 입력폼: 마스킹/재입력/오입력 방지
- 알림(푸시) 권한 요청은 "가치" 설명 후 요청
