# 프로젝트 문서 자동화 스킬 (Docs Generation)

이 스킬은 코드가 수정되어 API 시그니처, 클래스/메서드 구조, 동작 방식이 변경되었을 때 관련 문서를 최신 상태로 동기화(Sync)하기 위해 사용됩니다.

## 🎯 주요 목적
- 코드와 문서 간의 불일치(Outdated Documentation) 방지
- 복잡한 라이브러리 및 API 사용 가이드 최신화
- 마크다운(Markdown) 기반 문서 주기적 갱신

## 🚀 실행 지침

1. **문서 갱신 트리거**
   - 사용자가 새로운 클래스, 모듈, API 엔드포인트를 추가하거나 기존 로직의 인터페이스(파라미터, 반환값, 응답 형식 등)를 수정했을 때 발동합니다.
   - 주요 작업이 끝난 후 `VERIFICATION` 단계에서 문서화 누락 여부를 판단합니다.

2. **자동화 수행 절차**
   - **변경점 분석**: `git diff` 혹은 변경된 파일들을 탐색하여 어떤 기능이 추가/수정되었는지 요약합니다.
   - **대상 문서 탐색**: 프로젝트 내의 `docs/`, `README.md`, 또는 `apps/web/README.md` 등 관련된 문서 위치를 파악합니다.
   - **문서 작성/수정**: 변경된 소스 코드의 인라인 주석(Docstring 등)과 실제 동작 로직(인자, 반환 구조)을 기반으로 문서를 업데이트합니다. 
   - HWP 자동화 매뉴얼 작성 케이스처럼 복잡한 모듈이라면, 클래스와 메서드 목록을 트리 형태로 정리하고, 각 요소의 역활을 Markdown 표나 리스트로 일목요연하게 작성하여 반영합니다.

3. **작성 가이드라인**
   - 문서 업데이트 시 기존 스타일 가이드라인(한국어/영어, 존댓말/반말, 표 구성 방식 등)을 그대로 유지합니다.
   - 단순히 코드 복사 붙여넣기가 아니라, '사용자 관점'에서 어떻게 해당 API를 호출해야 하는지 예제 코드(Snippet)를 포함하여 구성합니다.

4. **결과 피드백**
   - 문서가 성공적으로 업데이트되었다면 `walkthrough.md` 또는 `task Summary`에 문서가 함께 업데이트 되었음을 명시합니다.
