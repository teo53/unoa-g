# 데이터 시각화 및 리포팅 스킬 (Data Viz)

이 스킬은 텍스트나 숫자 형태로만 존재하는 로그/결과 데이터를 AI 분석 또는 리포팅 목적으로 시각화(차트, 대시보드, PNG 등)하는 데 사용됩니다.

## 🎯 주요 목적
- Python 및 프론트엔드 환경에서의 데이터 시각화 스크립트 작성 최적화
- 데이터 분석 결과를 직관적으로 이해할 수 있는 리포트(이미지/PDF 등) 파일 생성
- `matplotlib`, `plotly`, `seaborn` 등 라이브러리 활용 표준화

## 🚀 실행 지침

1. **상황 인식 및 데이터 확인**
   - 데이터 시각화 요청이 들어오면, 가장 먼저 입력이 될 데이터 소스(CSV, JSON, DB 질의 결과, 혹은 로그 파일)의 형식과 샘플을 조회(`view_file` 혹은 `run_command`로 스크립트 실행)하여 파악합니다.

2. **시각화 스크립트 패턴 작성**
   - Python 환경의 경우 아래 요소가 포함된 전용 스크립트(예: `tools/generate_report.py`)를 구성합니다:
     - 한글 폰트 깨짐 방지 코드 (예: `plt.rc('font', family='Malgun Gothic')` 또는 `AppleGothic` 분기 처리)
     - 고해상도(DPI 300 이상) 이미지 출력 설정
     - 데이터 클렌징(전처리) -> 그래프 생성(Bar, Line, Pie, Scatter 등) -> `analysis_report.png`와 같은 파일로 저장 로직 연계
   
3. **템플릿 활용**
   - 사용자에게 반복적으로 보고할 대시보드 템플릿이라면 구조화된 서브플롯(Subplot)을 사용하여 여러 그래프를 1개의 이미지 버퍼 레이아웃에 배치합니다.
   - 축 레이블, 범례, 타이틀(Title)은 누락 없이 반드시 기재합니다.

4. **코드 실행 및 산출물 보관**
   - 생성 스크립트를 `run_command`로 실행한 뒤 지정된 디렉토리(예: `docs/reports/` 혹은 아티팩트 경로)에 이미지가 단일 파일로 제대로 떨어지는지 확인합니다.
   - 이미지가 생성되면 사용자에게 해당 파일 경로를 알려주고, 필요에 따라 `walkthrough.md`에 `![Report](file:///경로)` 형태로 삽입하여 보여줍니다.
