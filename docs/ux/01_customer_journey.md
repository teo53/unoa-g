# 01. Customer Journey Map

> UNO A -- Fan & Creator 여정 지도
> 최종 수정: 2026-02-08

---

## 1. Fan Journey

### Stage 1: Acquisition (발견)

| 항목 | 내용 |
|------|------|
| **터치포인트** | SNS 공유 링크, 앱스토어 검색, 아티스트 프로필 페이지 직접 유입 |
| **행동** | 앱 설치 / 웹 접속 -> 홈 화면 탐색 -> `/discover`에서 아티스트 검색 |
| **감정** | 호기심, 기대감 ("내 최애가 여기 있을까?") |
| **마찰** | 첫 방문 시 `/login`으로 강제 이동 -> 탐색 불가 (Friction #1) |
| **기회** | 비로그인 상태에서 홈 + 탐색 + 아티스트 프로필 열람 허용 -> 가치를 먼저 경험 |

### Stage 2: Activation (가입/첫 경험)

| 항목 | 내용 |
|------|------|
| **터치포인트** | 회원가입 화면, 데모 모드, 첫 구독 |
| **행동** | 계정 생성 -> 아티스트 검색 -> 프로필 확인 -> BASIC 구독 |
| **감정** | 설렘 ("드디어 연결됐다!"), 약간의 불안 ("돈 내야 하나?") |
| **마찰** | 회원가입 14개 입력 필드 한 페이지 (Friction #9), 토큰 시스템 설명 부재 (Friction #8) |
| **기회** | 2단계 Progressive Registration, 첫 구독 시 무료 체험 기간, 토큰 안내 바텀시트 |

### Stage 3: Transaction (결제/전송)

| 항목 | 내용 |
|------|------|
| **터치포인트** | DT 충전 (`/wallet/charge`), 구독 결제, 후원 메시지 전송 |
| **행동** | DT 충전 -> 아티스트 구독 -> 브로드캐스트 수신 -> 답글 토큰 사용 -> 후원 |
| **감정** | 만족감 ("아티스트가 내 메시지를 읽겠지"), 긴장감 ("토큰 3개밖에 없는데") |
| **마찰** | DM 전송 실패 시 제네릭 에러 (Friction #2), 펀딩 결제 silent fail (Friction #3) |
| **기회** | 실패 시 명확한 이유 + 다음 행동 안내, auth gate로 로그인 유도 |

### Stage 4: Retention (재방문)

| 항목 | 내용 |
|------|------|
| **터치포인트** | 푸시 알림, 새 브로드캐스트, 구독 갱신 안내 |
| **행동** | 알림 수신 -> 앱 진입 -> 새 메시지 확인 -> 답글 -> (반복) |
| **감정** | 친밀감 ("아티스트와 대화하는 느낌"), 충성 ("구독 유지해야지") |
| **마찰** | Bottom nav 배지가 선택 시에만 표시 -> 새 메시지 인지 불가 (Friction #10) |
| **기회** | 구독 일수 기반 글자 수 증가로 장기 구독 보상, 미읽은 메시지 배지 상시 표시 |

## 2. Creator Journey

### Stage 1: Acquisition (발견)

| 항목 | 내용 |
|------|------|
| **터치포인트** | 크리에이터 모집 페이지, 기존 팬 플랫폼에서 전환 안내 |
| **행동** | 플랫폼 정보 확인 -> 크리에이터 계정 신청 -> 승인 대기 |
| **감정** | 기대감 ("새로운 수익 채널"), 비교심 ("Fromm/Bubble보다 나은가?") |
| **마찰** | 크리에이터 전환 프로세스 불명확 |
| **기회** | 예상 수익 시뮬레이터, 기존 플랫폼 데이터 이전 지원 |

### Stage 2: Activation (채널 개설)

| 항목 | 내용 |
|------|------|
| **터치포인트** | 프로필 설정 (`/creator/content`), 대시보드 첫 진입 |
| **행동** | 프로필 사진/배너 설정 -> 채널 소개 작성 -> 구독 티어 설정 -> 첫 브로드캐스트 |
| **감정** | 성취감 ("내 채널이다!"), 약간의 부담감 ("뭘 써야 하지?") |
| **마찰** | 대시보드 9섹션 과밀 -> 정보 과부하 (Friction #7) |
| **기회** | 섹션 접기/펼치기 Progressive Disclosure, 첫 브로드캐스트 템플릿 제공 |

### Stage 3: Transaction (수익 창출)

| 항목 | 내용 |
|------|------|
| **터치포인트** | 브로드캐스트 전송, 팬 답글 확인, DT 수익 확인, 펀딩 캠페인 |
| **행동** | 메시지 작성 -> 전체 전송 -> 팬 반응 확인 -> 하트 반응 -> 수익 대시보드 확인 |
| **감정** | 보람 ("팬들이 반응해준다"), 동기부여 ("수익이 올라간다") |
| **마찰** | CRM 데이터와 채팅의 분리, 개별 팬 히스토리 추적 어려움 |
| **기회** | 대시보드에서 팬 세그먼트별 통계 제공, AI 답글 초안 기능 |

### Stage 4: Retention (지속 운영)

| 항목 | 내용 |
|------|------|
| **터치포인트** | 정기 수익 리포트, 팬 이탈 알림, 채널 성장 지표 |
| **행동** | 주간/월간 통계 확인 -> 콘텐츠 전략 조정 -> 특별 이벤트 기획 |
| **감정** | 안정감 ("수익이 꾸준하다"), 성장 욕구 ("더 키우고 싶다") |
| **마찰** | 장기 운영 시 콘텐츠 피로감, 팬 이탈 원인 분석 도구 부재 |
| **기회** | AI 기반 콘텐츠 제안, 이탈 위험 팬 사전 알림, 크리에이터 커뮤니티 |

## 3. Journey 핵심 지표

| 단계 | Fan KPI | Creator KPI |
|------|---------|-------------|
| Acquisition | 앱 설치 수, 탐색 페이지 도달률 | 크리에이터 신청 수 |
| Activation | 회원가입 완료율, 첫 구독 전환율 | 첫 브로드캐스트 전송률 |
| Transaction | ARPU, 답글 토큰 사용률, DT 충전 금액 | 월간 수익, 팬당 후원 금액 |
| Retention | 30일 재방문율, 구독 갱신율 | 주간 브로드캐스트 빈도, 팬 이탈률 |
