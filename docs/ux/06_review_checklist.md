# 06. QA Review Checklist

> UNO A -- 플로우별 QA 체크리스트
> 최종 수정: 2026-02-08

---

## 1. Guest Flow (비로그인 사용자)

### 1.1 첫 진입 및 탐색

- [ ] `/` (홈) 진입 시 가치 제안 배너 표시
- [ ] `/discover` 접근 가능, 아티스트 리스트 표시
- [ ] `/funding` 접근 가능, 캠페인 리스트 표시
- [ ] `/artist/:artistId` 프로필 열람 가능 (구독 티어, 소개)
- [ ] 하단 네비게이션 표시 (홈, 메시지, 펀딩, 탐색, 프로필)
- [ ] 로그인 없이 탐색, 검색, 프로필 조회 가능

### 1.2 행동 게이트 (Auth Gate)

- [ ] 메시지 전송 시도 -> Auth Gate 바텀시트 표시
- [ ] 구독 버튼 탭 -> Auth Gate 바텀시트 표시
- [ ] 펀딩 결제 시도 -> Auth Gate 바텀시트 표시
- [ ] Auth Gate에 "로그인", "회원가입", "데모 체험" 3개 옵션 존재
- [ ] "로그인" 선택 -> `/login` 이동
- [ ] "회원가입" 선택 -> `/register` 이동
- [ ] "데모 체험" 선택 -> 데모 모드 진입, 원래 행동 계속 가능
- [ ] Auth Gate 닫기 (dismiss) 가능, 화면 유지

### 1.3 프로필 (비로그인)

- [ ] `/profile` 접근 시 게스트 뷰 표시 (MockData 노출 금지)
- [ ] 게스트 뷰에 로그인 유도 CTA 표시
- [ ] 가짜 사용자 정보 (이름, 잔액 등) 미표시

## 2. Fan Flow (로그인 사용자)

### 2.1 가입 및 첫 경험

- [ ] 회원가입 2단계 분리 (기본정보 -> 선호도)
- [ ] Step 1 완료 후 Step 2로 자연스러운 전환
- [ ] 회원가입 완료 -> 홈 화면 이동
- [ ] 홈 화면에 추천 아티스트 또는 온보딩 배너 표시

### 2.2 아티스트 탐색 및 구독

- [ ] `/discover` 아티스트 검색 (이름, 카테고리)
- [ ] 아티스트 프로필 조회 (배너, 소개, 구독 티어)
- [ ] 구독 버튼 탭 -> 티어 선택 (BASIC / STANDARD / VIP)
- [ ] 구독 완료 -> 성공 토스트 표시
- [ ] 구독 후 채팅 리스트에 아티스트 추가

### 2.3 채팅 (1:1 환상)

- [ ] `/chat` 구독 아티스트 채팅 리스트 표시
- [ ] 미읽은 메시지 배지 표시 (Bottom nav + 채팅 리스트)
- [ ] 채팅 진입 -> 내 메시지 + 아티스트 메시지만 표시
- [ ] 아티스트 브로드캐스트 수신 -> 1:1 대화처럼 보임
- [ ] 답글 토큰 잔여량 표시
- [ ] 첫 채팅 진입 시 토큰 설명 바텀시트 표시 (Friction #8 해결)
- [ ] 답글 작성 -> 토큰 차감 -> 전송 완료 토스트
- [ ] 토큰 0개 -> 전송 불가 + "다음 브로드캐스트 시 충전됩니다" 안내
- [ ] 구독 일수에 따른 글자 수 제한 적용 (50-300자)

### 2.4 후원 및 DT

- [ ] DT 잔액 확인 (`/wallet`)
- [ ] DT 구매 (`/wallet/charge`) -> 금액 선택 -> 결제
- [ ] 후원 메시지 전송 -> DT 차감 -> 성공 토스트
- [ ] 잔액 부족 시 안내 메시지 표시

### 2.5 펀딩

- [ ] `/funding` 캠페인 리스트 표시
- [ ] 캠페인 상세 조회 (목표액, 진행률, 리워드)
- [ ] 펀딩 참여 -> 금액 입력 -> 결제 -> 완료 토스트
- [ ] 결제 실패 시 명확한 에러 메시지 (silent fail 금지)

## 3. Creator Flow

### 3.1 대시보드

- [ ] `/creator/dashboard` 진입 시 CRM 통합 대시보드 표시
- [ ] 섹션 접기/펼치기 동작 (Progressive Disclosure)
- [ ] 구독자 수, 월간 수익, 메시지 통계 표시
- [ ] 각 섹션 정보 정확성 확인

### 3.2 채팅 (단체톡방)

- [ ] `/creator/chat` 2탭 구조 (내 채널 / 구독 아티스트)
- [ ] 내 채널 탭: 모든 팬 메시지 시간순 통합 표시
- [ ] 팬 메시지: 왼쪽 정렬 + 팬 이름 + 구독 티어 배지
- [ ] 크리에이터 메시지: 오른쪽 정렬 + "전체" 라벨
- [ ] 메시지 입력 -> 모든 팬에게 전송 (브로드캐스트)
- [ ] 팬 메시지에 하트 반응 버튼 동작
- [ ] 구독 아티스트 탭: 팬으로서의 1:1 채팅 리스트

### 3.3 프로필 관리

- [ ] `/creator/profile` 프로필 정보 표시
- [ ] `/creator/content` 프로필 편집 동작
- [ ] 배너/아바타 변경 -> 미리보기 -> 저장
- [ ] 테마 색상 변경 (6가지 프리셋)

### 3.4 펀딩 관리

- [ ] `/creator/funding` 내 캠페인 리스트 표시
- [ ] `/creator/funding/create` 캠페인 생성 플로우
- [ ] 캠페인 편집 (`/creator/funding/edit/:id`) 동작

## 4. Accessibility Checklist

### 4.1 터치 타겟

- [ ] 모든 인터랙티브 요소 최소 48x48dp
- [ ] 버튼 간 충분한 간격 (최소 8dp)
- [ ] `AccessibleTapTarget` 래퍼 사용 확인

### 4.2 색상 대비

- [ ] 텍스트 대비 4.5:1 이상 (WCAG AA)
- [ ] `primary600` (#DE332A) + `onPrimary` (#FFFFFF) 대비 확인
- [ ] `danger` (#B42318) + white 대비 확인
- [ ] `textMuted` (#6B7280) on `background` (#F8F8F8) 대비 확인
- [ ] Dark 모드에서 동일 대비 기준 충족

### 4.3 Semantic Labels

- [ ] 모든 아이콘 버튼에 `semanticLabel` 설정
- [ ] 이미지에 `SemanticImage` 래퍼 사용
- [ ] 스크린 리더 순서 논리적 (탭 순서 = 시각 순서)
- [ ] 상태 변경 시 `ScreenReaderAnnouncement.announce()` 호출
- [ ] 장식용 요소 `.excludeSemantics()` 처리

### 4.4 반응형

- [ ] 폰 (< 600px): 단일 컬럼
- [ ] 태블릿 (600-1024px): 적절한 여백 또는 2컬럼
- [ ] 웹: `AppScaffold`으로 폰 프레임 UI
- [ ] 가로 모드에서 레이아웃 깨짐 없음
- [ ] `AdaptiveContainer` 사용으로 큰 화면 센터링

## 5. Cross-Cutting Concerns

### 5.1 다크 모드

- [ ] 설정 -> 다크 모드 토글 -> 전체 앱 전환
- [ ] 모든 화면에서 다크 모드 색상 정상 적용
- [ ] 카드 테두리, 배경, 텍스트 색상 전환 확인
- [ ] 이미지/아바타 다크 모드에서 적절한 대비

### 5.2 데모 모드

- [ ] 데모 모드 진입 -> `DemoConfig` 데이터 사용
- [ ] 하드코딩된 ID/URL/금액 없음 (`DemoConfig.*` 사용)
- [ ] 데모 모드에서 모든 핵심 플로우 체험 가능
- [ ] 데모 모드 해제 -> 정상 상태 복귀

### 5.3 에러 처리

- [ ] 네트워크 오류 -> `ErrorDisplay` + 재시도 버튼
- [ ] 데이터 없음 -> `EmptyState` + 행동 유도
- [ ] 로딩 중 -> `SkeletonLoader` (CLS 방지)
- [ ] API 실패 시 사용자 친화적 메시지 (개발자 에러 미노출)

### 5.4 성능

- [ ] 초기 로딩 3초 이내
- [ ] 화면 전환 애니메이션 끊김 없음
- [ ] 채팅 스크롤 부드러움 (100+ 메시지)
- [ ] 이미지 lazy loading 적용
