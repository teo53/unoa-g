# 02. 사용자 여정 및 UX 갭 (User Journeys & UX Gaps)

## 1. 사용자 유형

| 유형 | 설명 | 핵심 목표 |
|------|------|----------|
| **팬** | 아티스트 구독 및 소통 | 아티스트와 가까워지기, 특별한 경험 |
| **크리에이터** | 콘텐츠 제공 및 수익화 | 팬과 소통, 수익 창출 |
| **어드민** | 플랫폼 운영 및 관리 | 안전한 커뮤니티 유지, 분쟁 해결 |

## 2. 팬 사용자 여정

### 2.1 온보딩 → 첫 구독

```
┌─────────────────────────────────────────────────────────────────────┐
│ 1. 앱 설치                                                          │
│    ├── 앱스토어에서 발견                                             │
│    ├── 설치 → 실행                                                  │
│    └── 스플래시 화면 (로고, 로딩)                                    │
│                                                                     │
│ 2. 회원가입                                                         │
│    ├── 소셜 로그인 선택 (카카오/구글/애플)                           │
│    ├── 약관 동의 화면 ⚠️ [현재 없음]                                │
│    ├── 연령 인증 (만 14세 이상) ✅                                   │
│    └── 닉네임/프로필 설정                                           │
│                                                                     │
│ 3. 홈 화면                                                          │
│    ├── 추천 아티스트 목록 ✅                                         │
│    ├── 트렌딩 아티스트 ✅                                            │
│    └── 검색 기능 ✅                                                  │
│                                                                     │
│ 4. 아티스트 발견                                                     │
│    ├── 탐색 탭 ✅                                                    │
│    ├── 아티스트 프로필 조회 ✅                                       │
│    └── 구독 티어 확인 ✅                                             │
│                                                                     │
│ 5. 첫 구독                                                          │
│    ├── 구독 버튼 클릭                                               │
│    ├── 티어 선택 (BASIC/STANDARD/VIP) ✅                            │
│    ├── 결제 화면 ⚠️ [UI만 존재, 연동 미완]                          │
│    └── 구독 완료 → 채팅방 입장                                       │
└─────────────────────────────────────────────────────────────────────┘
```

**UX 갭**:
- ❌ 약관 동의 화면 없음 (법적 필수)
- ⚠️ 결제 플로우 미완성
- ❌ 온보딩 튜토리얼 없음
- ❌ 푸시 알림 권한 요청 타이밍 불명확

## 5. 상태별 UI 점검

### 5.1 빈 상태 (Empty State)

| 화면 | 현재 상태 | 필요 조치 |
|------|----------|----------|
| 채팅 목록 (구독 없음) | ⚠️ 기본 텍스트 | EmptyState 위젯 적용 |
| 채팅 메시지 (첫 방문) | ⚠️ 없음 | 환영 메시지 추가 |
| 알림 목록 | ⚠️ 없음 | EmptyState 위젯 적용 |
| 거래 내역 | ⚠️ 없음 | EmptyState 위젯 적용 |
| 펀딩 목록 | ⚠️ 없음 | EmptyState 위젯 적용 |
| 검색 결과 | ⚠️ 없음 | EmptyState 위젯 적용 |

### 5.2 에러 상태 (Error State)

| 화면 | 현재 상태 | 필요 조치 |
|------|----------|----------|
| 네트워크 오류 | ⚠️ 기본 에러 | ErrorBoundary + 재시도 |
| 서버 오류 | ⚠️ 기본 에러 | 사용자 친화적 메시지 |
| 인증 만료 | ⚠️ 없음 | 재로그인 유도 |
| 권한 오류 | ⚠️ 없음 | 명확한 안내 |

### 5.3 로딩 상태 (Loading State)

| 화면 | 현재 상태 | 필요 조치 |
|------|----------|----------|
| 초기 로딩 | ✅ 스플래시 | - |
| 목록 로딩 | ✅ Skeleton | - |
| 상세 로딩 | ⚠️ 기본 | Skeleton 적용 |
| 결제 처리 중 | ⚠️ 없음 | 로딩 오버레이 |
| 전송 중 | ⚠️ 없음 | 전송 중 표시 |

### 5.4 권한 없음 상태

| 화면 | 현재 상태 | 필요 조치 |
|------|----------|----------|
| 미구독 채팅방 | ✅ 구독 유도 | - |
| 크리에이터 전용 기능 | ⚠️ 없음 | 안내 + 전환 유도 |
| 어드민 전용 기능 | ⚠️ 없음 | 404 또는 리다이렉트 |

## 8. 결제 UX 점검 (이탈 포인트)

```
┌─────────────────────────────────────────────────────────────────────┐
│ 결제 퍼널                                                           │
│                                                                     │
│ 1. 상품 선택 → 2. 결제 수단 → 3. 결제 진행 → 4. 완료                │
│      ✅             ⚠️            ❌           ❌                   │
│                                                                     │
│ 이탈 포인트:                                                        │
│ • 결제 수단 선택 화면 복잡도                                         │
│ • 결제 진행 중 로딩 상태 표시 부재                                   │
│ • 결제 실패 시 명확한 안내 부재                                      │
│ • 결제 취소 시 돌아갈 곳 불명확                                      │
└─────────────────────────────────────────────────────────────────────┘
```

**개선 필요**:
1. 결제 수단 간소화 (저장된 카드 우선)
2. 결제 진행 중 프로그레스 표시
3. 결제 실패 시 명확한 사유 + 재시도 버튼
4. 결제 완료 후 성공 애니메이션
