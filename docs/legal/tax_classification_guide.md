# UNO A 서비스 과세 분류 가이드 (총매법 모델)

## 1. 사업 모델 및 법적 지위

### 1.1 회사 지위: 통신판매업자 (총매법/Principal-Gross Sales Model)

| 항목 | 내용 |
|------|------|
| **법적 지위** | 통신판매업자 (전자상거래법 제12조) |
| **매출 인식** | 총액 기준 (Gross Revenue) |
| **DT 법적 성격** | 서비스 전용 디지털 이용권 (선불전자지급수단 비해당) |
| **크리에이터 관계** | 광고용역 제공자 (독립 계약자) |
| **적용 법체계** | 전자상거래법, 콘텐츠산업진흥법, 소득세법, 부가가치세법 |

> **핵심**: 회사는 "중개자"가 아닌 "직접 판매자"입니다. DT는 전자금융거래법상 선불전자지급수단에 해당하지 않으며, 전자지급업 등록 대상이 아닙니다.

### 1.2 DT가 선불전자지급수단이 아닌 근거

1. **양도 불가**: 사용자 간 DT 이전 기술적/정책적으로 차단
2. **서비스 내 전용**: 회사가 제공하는 디지털 서비스에만 사용 가능
3. **현금화 불가**: 환전, 출금, 외부 거래 불가
4. **제3자 결제수단 아님**: 크리에이터에게 결제대금을 이전하거나 지급하는 수단이 아님

## 2. 결제 방식 분리

UNO A는 두 가지 결제 방식을 사용합니다:

| 결제 방식 | 서비스 유형 | 레퍼런스 모델 |
|-----------|-------------|---------------|
| **DT (Dream Time)** | 구독, 후원, 유료답장 | Fromm, Bubble |
| **원화 (KRW)** | 펀딩, 굿즈 상품 | Makestar |

## 3. DT 기반 서비스 과세 (구독/후원/유료답장)

### 3.1 DT 구매 (User → Platform) — 2단계 수익 인식

| 단계 | 시점 | 회계 처리 | VAT |
|------|------|----------|-----|
| **1단계: DT 구매** | 결제 완료 시 | 계약부채(선수금) 인식 | 과세 아님 |
| **2단계: DT 사용** | 서비스 제공 시 | 매출 인식 | VAT 과세(공급시기) |
| **환불** | 환불 처리 시 | 계약부채 차감 | 매출 미인식 |
| **소멸(5년)** | 유효기간 만료 시 | 매출 인식 | VAT 과세 |

> **중요**: DT 구매 시점에는 매출이 아닌 선수금(계약부채)으로 인식합니다. 실제 서비스 제공(DT 사용) 시점이 매출 인식 및 VAT 과세 시점입니다.

### 3.2 DT 사용 시점 과세

DT 사용 시 **서비스 제공 완료** 시점에 과세됩니다.

#### 구독료 (Subscription)

| 항목 | 내용 |
|------|------|
| **서비스 유형** | 디지털 콘텐츠 구독 (Bubble/Fromm 모델) |
| **과세 구분** | 부가가치세 과세 |
| **세율** | 10% |
| **소득 분류** | 크리에이터에 대한 광고용역비 지급 |
| **정산 시 원천징수** | 아래 3.3 참조 |

#### 후원/팁 (Donation/Tip)

| 항목 | 내용 |
|------|------|
| **서비스 유형** | 크리에이터 응원 (투네이션/슈퍼챗 유사) |
| **회사 입장** | 총액 매출 인식 (총매법) |
| **크리에이터 입장** | 회사로부터 광고용역비 수취 |
| **비고** | "소비자 대금 전달"이 아닌 "회사 비용(광고용역비) 지급" |

#### 유료 답장 (Paid Reply)

| 항목 | 내용 |
|------|------|
| **서비스 유형** | 디지털 콘텐츠 용역 |
| **과세 구분** | 부가가치세 10% 과세 |
| **회사 입장** | 총액 매출 인식 |
| **크리에이터 입장** | 광고용역비 수취 |

### 3.3 크리에이터 정산 원천징수

크리에이터에 대한 정산은 "소비자 결제대금의 전달"이 아니라, 회사가 별도 기준으로 산정해 지급하는 **광고용역비**입니다.

| 크리에이터 유형 | 원천징수율 | 비고 |
|----------------|-----------|------|
| **개인사업자** | 3.3% (사업소득) | 사업자등록 필요 |
| **법인** | 없음 | 법인세 자체 신고 |
| **비사업자 (프리랜서)** | 8.8% (기타소득) | 필요경비 60% 공제 후 22% |

## 4. KRW 기반 서비스 과세 (펀딩/굿즈)

### 4.1 크라우드펀딩 (Crowdfunding)

| 항목 | 내용 |
|------|------|
| **결제 수단** | 신용카드, 계좌이체, 간편결제 (KRW) |
| **과세 기준** | **총 결제금액** 기준 (플랫폼 수수료 공제 전) |
| **부가세** | 10% (리워드 제공이 있는 보상형 펀딩) |
| **소득 분류** | 크리에이터 사업소득 |
| **참고** | 와디즈/텀블벅 방식과 동일, 국세청 분기별 매출자료 제출 의무 |

> **펀딩 실패 시**: 목표금액 미달 시 전액 환불, 부가세 발생 안 함

### 4.2 굿즈 상품 판매 (Goods Store)

| 항목 | 내용 |
|------|------|
| **결제 수단** | 신용카드, 계좌이체, 간편결제 (KRW) |
| **법적 성격** | 재화 판매 (통신판매업) |
| **부가세** | 10% |
| **통신판매업 신고** | 필요 (연 50회 이상 거래 또는 일반과세자) |

## 6. 정산 및 원천징수

### 6.1 DT 서비스 정산 공식

```
총 매출 = DT 사용 기준 매출 인식액 (구독+후원+유료답장)
플랫폼 수수료 = 총 매출 × 20%
크리에이터 광고용역비 = 총 매출 × 80%
원천징수 세액:
  - 개인사업자: 3.3% (사업소득)
  - 비사업자: 8.8% (기타소득)
  - 법인: 없음
순 정산 지급액 = 광고용역비 - 원천징수
```

### 6.2 KRW 서비스 정산 공식

```
총 수입 = 펀딩 후원금 + 굿즈 매출 (KRW)
플랫폼 수수료 = 총 수입 × 10~15%
결제대행 수수료 = 총 결제액 × 2.5~3.5%
부가세 = 공급가액 × 10%
순 정산 지급액 = 총 수입 - 수수료 - 결제수수료 - 부가세
```

## 7. 환불 처리

| 항목 | 내용 |
|------|------|
| **적용 법률** | 전자상거래법 (전자금융거래법 비해당) |
| **환불 기한** | 3영업일 이내 환급 |
| **지연배상금** | 대통령령이 정하는 지연이자율 적용 |
| **DT 미사용분** | 구매일로부터 7일 이내 전액 환불 |
| **DT 일부 사용** | 잔량 부분환불 허용 |
| **회계 처리** | 환불 시 계약부채(선수금) 차감 — 매출 미인식 |

> **현행 지연배상금 이율**: 연 20% (전자상거래법 시행령 제21조의2)
> ⚠️ UI/약관에는 "대통령령이 정하는 지연이자율"로만 표기. 구체 이율은 시행령 개정 시 변경될 수 있으므로, 이 내부 문서에만 버전 관리합니다.
> **문서 버전**: 2026-02-16 기준
