## 1. 결제 방식 분리

UNO A는 두 가지 결제 방식을 사용합니다:

| 결제 방식 | 서비스 유형 | 레퍼런스 모델 |
|-----------|-------------|---------------|
| **DT (Dream Time)** | 구독, 후원, 유료답장 | Fromm, Bubble |
| **원화 (KRW)** | 펀딩, 굿즈 상품 | Makestar |

---

## 2. DT 기반 서비스 과세 (구독/후원/유료답장)

### 2.1 DT 충전 (User → Platform)

| 항목 | 내용 |
|------|------|
| **법적 성격** | 선불전자지급수단 발행 (전자금융거래법) |
| **부가세** | 충전 시점에 과세 안 함 |
| **비고** | 환불 가능 (미사용분에 한함) |

### 2.2 DT 사용 시점 과세

DT 사용 시 **서비스 제공 완료** 시점에 과세됩니다.

#### 구독료 (Subscription)

| 항목 | 내용 |
|------|------|
| **서비스 유형** | 디지털 콘텐츠 구독 (Bubble/Fromm 모델) |
| **과세 구분** | 부가가치세 과세 |
| **세율** | 10% |
| **소득 분류** | 크리에이터 사업소득 |
| **정산 시 원천징수** | 3.3% (개인/프리랜서) |

#### 후원/팁 (Donation/Tip)

| 항목 | 내용 |
|------|------|
| **서비스 유형** | 크리에이터 응원 (투네이션/슈퍼챗 유사) |
| **과세 구분** | 기타소득 (대가성 없는 후원) |
| **세율** | 22% (지방소득세 포함, 필요경비 60% 공제 후) |
| **원천징수** | 8.8% (= 22% × 40%) |
| **비고** | 건당 5만원 이하 분리과세 가능 |

#### 유료 답장 (Paid Reply)

| 항목 | 내용 |
|------|------|
| **서비스 유형** | 디지털 콘텐츠 용역 |
| **과세 구분** | 부가가치세 10% 과세 |
| **소득 분류** | 사업소득 또는 기타소득 |
| **원천징수** | 3.3% (사업소득 처리 시) |

## 3. KRW 기반 서비스 과세 (펀딩/굿즈)

### 3.1 크라우드펀딩 (Crowdfunding)

| 항목 | 내용 |
|------|------|
| **결제 수단** | 신용카드, 계좌이체, 간편결제 (KRW) |
| **과세 기준** | **총 결제금액** 기준 (플랫폼 수수료 공제 전) |
| **부가세** | 10% (리워드 제공이 있는 보상형 펀딩) |
| **소득 분류** | 크리에이터 사업소득 |
| **세금계산서** | 크리에이터가 발행 가능 (사업자인 경우) |
| **참고** | 와디즈/텀블벅 방식과 동일, 국세청 분기별 매출자료 제출 의무 |

> **펀딩 실패 시**: 목표금액 미달 시 전액 환불, 부가세 발생 안 함

### 3.2 굿즈 상품 판매 (Goods Store)

| 항목 | 내용 |
|------|------|
| **결제 수단** | 신용카드, 계좌이체, 간편결제 (KRW) |
| **법적 성격** | 재화 판매 (통신판매업) |
| **부가세** | 10% |
| **소득 분류** | 사업소득 |
| **세금계산서** | 발행 가능 (일반과세자) |
| **통신판매업 신고** | 필요 (연 50회 이상 거래 또는 일반과세자) |

> **사업자 유형**:
> - 일반과세자: 연매출 8000만원↑, 10% 부가세, 매입세액 공제
> - 간이과세자: 연매출 4800만원~8000만원, 간소화된 부가세
> - 면세: 연매출 4800만원↓, 부가세 납부 면제

## 5. 정산 및 원천징수

### 5.1 DT 서비스 정산 공식

```
총 수입 = 구독료 + 후원금 + 유료답장 (DT)
플랫폼 수수료 = 총 수입 × 20%
원천징수 세액:
  - 사업소득(구독/유료답장): 3.3%
  - 기타소득(후원): 8.8%
순 정산액 = 총 수입 - 플랫폼 수수료 - 원천징수
```

### 5.2 KRW 서비스 정산 공식

```
총 수입 = 펀딩 후원금 + 굿즈 매출 (KRW)
플랫폼 수수료 = 총 수입 × 10~15%
결제대행 수수료 = 총 결제액 × 2.5~3.5%
부가세 = 공급가액 × 10%
순 정산액 = 총 수입 - 수수료 - 결제수수료 - 부가세
```

## 7. 구현 요구사항

### 7.1 데이터베이스 스키마

```sql
-- wallet_ledger: DT 거래 세금 분류
ALTER TABLE wallet_ledger ADD COLUMN tax_category VARCHAR(20);
-- 값: subscription, donation, paid_reply, refund

-- krw_transactions: 원화 거래 테이블 (신규)
CREATE TABLE krw_transactions (
  id UUID PRIMARY KEY,
  user_id UUID REFERENCES auth.users(id),
  type VARCHAR(20) NOT NULL, -- 'funding', 'goods'
  amount_krw INTEGER NOT NULL,
  vat_amount INTEGER NOT NULL, -- 10% VAT
  payment_method VARCHAR(20), -- 'card', 'bank', 'kakaopay' etc.
  status VARCHAR(20) DEFAULT 'pending',
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- creator_payouts: 정산 세금 정보
ALTER TABLE creator_payouts ADD COLUMN gross_amount INTEGER;
```
